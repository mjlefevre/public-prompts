# Role-Based Translation and Tone-Shifting Prompts

## QA (Quality Assurance)

1. "Translate this QA status update [text] into plain language for non-technical stakeholders while preserving risk and release implications."
2. "Rewrite this bug report [report] in a more neutral, factual tone that avoids blame."
3. "Convert this technical test result [result] into an executive summary focused on user impact and decision needed."
4. "Shift this QA feedback [feedback] from blunt to constructive while keeping the issues clear."
5. "Rewrite this release risk note [note] in a concise, professional tone for leadership."
6. "Translate this test plan [plan] into a checklist format that a new tester can follow."
7. "Convert this defect summary [summary] into customer-support-friendly language."
8. "Rewrite this blocked testing update [update] to sound calm, specific, and action-oriented."
9. "Translate this QA terminology [terms] into definitions suitable for business stakeholders."
10. "Condense this long testing update [update] into a short Slack message with status, risk, and next step."

## Business Analysts

1. "Translate this business requirement [requirement] into plain English while preserving rules, constraints, and exceptions."
2. "Rewrite this stakeholder email [email] in a more concise and executive-friendly tone."
3. "Convert this technical explanation [text] into business language focused on process, value, risk, and decision impact."
4. "Shift this requirements feedback [feedback] from critical to collaborative while keeping the requested changes explicit."
5. "Rewrite this workshop summary [summary] for an audience that did not attend the session."
6. "Translate this process description [process] into a step-by-step operational checklist."
7. "Convert this RFP response [response] into a more persuasive and evidence-based tone."
8. "Rewrite this user story [story] so it is clearer for developers, testers, and stakeholders."
9. "Translate this stakeholder concern [concern] into a neutral problem statement and decision question."
10. "Condense this discovery document [doc] into a one-page summary for leadership."

## Developers

3. "Convert this code comment [comment] into a more accurate and maintainable explanation."
4. "Shift this engineering update [update] from informal to professional while keeping it direct."
5. "Rewrite this incident note [note] in a neutral, factual tone suitable for a postmortem."
6. "Translate this API behavior [behavior] into customer-facing documentation language."
7. "Convert this debugging explanation [explanation] into a structured checklist for another developer."
8. "Rewrite this error message [message] so it is more helpful, specific, and user-safe."
9. "Translate this performance trade-off [tradeoff] into business impact and user experience terms."
10. "Condense this technical design [design] into a short summary for a cross-functional meeting."
