# Role-Based Transformation and Format Conversion Prompts

## QA (Quality Assurance)

1. "Convert these QA notes [notes] into a structured test plan with scope, scenarios, data needs, risks, and exit criteria."
2. "Turn this bug report [report] into a developer-ready issue template with all required fields."
3. "Convert this regression checklist [checklist] into a risk-based testing matrix."
4. "Transform this release testing summary [summary] into an executive status update."
5. "Turn this requirements document [doc] into test scenarios grouped by feature area and risk."
6. "Convert this test execution log [log] into a pass/fail/blocker dashboard summary."
7. "Transform these exploratory notes [notes] into reproducible bug reports and follow-up questions."
8. "Turn this QA retrospective [retro] into action items with owners, due dates, and success measures."
9. "Convert this UAT feedback [feedback] into defects, enhancement requests, questions, and sign-off risks."
10. "Transform this defect export [export] into a triage-ready table with severity, priority, owner, and next step."

## Business Analysts

1. "Convert this meeting transcript [transcript] into meeting minutes with decisions, action items, owners, and open questions."
2. "Turn this process narrative [narrative] into a swimlane-style step list by actor and system."
3. "Convert these stakeholder notes [notes] into requirements, assumptions, risks, dependencies, and decisions."
4. "Transform this business goal [goal] into capabilities, user stories, and acceptance criteria."
5. "Turn this workshop output [output] into a prioritized backlog with themes and next steps."
6. "Convert this requirements document [doc] into a traceability matrix."
7. "Transform this RFP text [text] into a response outline with owners, evidence needed, and deadlines."
8. "Turn this current-state description [description] into a gap analysis against the desired future state."
9. "Convert this spreadsheet-style field list [fields] into a data dictionary."
10. "Transform this stakeholder map [map] into a communication plan with cadence, channel, and decision rights."

## Developers

1. "Convert this pseudo-code [pseudo-code] into working code in [language/framework]."
2. "Turn this API description [description] into an OpenAPI-style endpoint specification."
3. "Convert this JSON sample [json] into typed models or interfaces in [language]."
4. "Transform this SQL query [query] into an optimized and readable version with explanations."
5. "Turn this shell workflow [workflow] into a reusable script with validation and error handling."
6. "Convert this callback-based code [code] into async/await style while preserving behavior."
7. "Transform this implementation discussion [discussion] into tasks, tests, and documentation updates."
8. "Turn this plain-language requirement [requirement] into technical acceptance criteria and test cases."
9. "Convert this error log [log] into a debugging checklist and likely root-cause hypotheses."
10. "Transform this code snippet [code] into a more idiomatic version for [language/framework]."

## Senior Developers

1. "Convert this architecture discussion [discussion] into an RFC with context, options, decision, risks, and rollout."
2. "Turn this incident timeline [timeline] into a postmortem with impact, causes, detection gaps, and action items."
3. "Transform this migration idea [idea] into a phased roadmap with rollback and validation steps."
4. "Convert this technical debt list [list] into a prioritized initiative plan."
5. "Turn this system description [description] into a component diagram narrative with dependencies and ownership."
6. "Transform this design review feedback [feedback] into required changes, optional suggestions, and open questions."
7. "Convert this operations runbook [runbook] into an incident response checklist."
8. "Turn this architecture proposal [proposal] into an executive decision memo."
9. "Transform this performance analysis [analysis] into hypotheses, experiments, measurements, and rollout criteria."
10. "Convert this cross-team plan [plan] into milestones, dependencies, owners, risks, and communication checkpoints."

## Product People

1. "Convert these customer interviews [notes] into themes, insights, opportunities, and evidence."
2. "Turn this product vision [vision] into roadmap themes, candidate bets, and measurable outcomes."
3. "Transform this feature idea [idea] into a PRD outline with problem, users, scope, metrics, and risks."
4. "Convert this feedback export [feedback] into a prioritized backlog using RICE or WSJF."
5. "Turn this launch plan [plan] into a cross-functional readiness checklist."
6. "Transform this roadmap discussion [discussion] into decisions, open questions, dependencies, and next steps."
7. "Convert this experiment result [result] into learnings, decision, follow-up experiments, and product implications."
8. "Turn this competitor research [research] into a positioning comparison table."
9. "Transform this product strategy [strategy] into a one-page memo for executives."
10. "Convert this user journey [journey] into pain points, opportunity areas, metrics, and experiment ideas."
